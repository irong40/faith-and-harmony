<objective>
Build the intake edge function that receives structured call data from n8n and creates client, quote request, and lead records, with co-located Deno unit tests.

Purpose: This is the core API for the voice bot pipeline. When a Vapi call ends, n8n transforms the data and POSTs it to this function. The function finds or creates a client by phone number, creates a quote request (feeding into the existing admin workflow), and creates a lead record linking the call to the request. Unit tests validate the pure logic (auth, validation, response shape) without requiring a live database per the project TDD mandate.
Output: Deployed edge function at /intake-lead accepting authenticated POST requests, plus passing Deno test suite.
</objective>

    Create `supabase/functions/intake-lead/index.ts` following the exact edge function pattern from `mission-control-api/index.ts` (see 01-RESEARCH.md Pattern 1, Pattern 2, Pattern 3).

    IMPORTANT: Structure the code so that pure logic functions are exported and testable. Export these functions/constants separately from the `serve()` call:
    - `validateWebhookSecret(headerValue: string | null, envSecret: string): boolean`
    - `validateRequiredFields(body: Record<string, unknown>): { valid: boolean; missing: string[] }`
    - `normalizePhone(phone: string): string`
    - `REQUIRED_FIELDS` constant array

    The function handles POST requests from n8n with this payload shape:
    ```typescript
    type IntakePayload = {
      caller_name: string;
      caller_phone: string;
      caller_email?: string;
      service_type: string;       // re_basic | re_standard | re_premium | construction | commercial | inspection
      job_description: string;
      call_id: string;            // Vapi call ID for linking to vapi_call_logs
      property_address?: string;  // optional, voice calls may not capture
      preferred_date?: string;    // optional, ISO date string
      qualification_status?: string; // qualified | declined | transferred
      sentiment?: string;         // positive | neutral | negative | unknown
    };
    ```

    Implementation steps in order:

    1. CORS preflight handling (OPTIONS returns 200 with standard headers).
    2. Method check: POST only, return 405 for others.
    3. Webhook secret validation: Compare `req.headers.get('x-webhook-secret')` against `Deno.env.get('INTAKE_WEBHOOK_SECRET')`. Return 401 if mismatch. This prevents unauthorized calls.
    4. Parse JSON body. Validate required fields (caller_name, caller_phone, service_type, job_description, call_id). Return 400 with specific error if any missing.
    5. Create Supabase client with SUPABASE_SERVICE_ROLE_KEY (not anon key, this bypasses RLS).
    6. Find or create client by phone number:
       - Normalize phone by stripping non-digits: `phone.replace(/\D/g, '')`
       - Query clients table for exact match on normalized phone OR raw phone
       - If found: use existing client_id, do not update existing record
       - If not found: insert new client with name, phone, email (if provided). Set created_by to null (system created).
    7. Insert into quote_requests with:
       - name: caller_name
       - email: caller_email or null (column is now nullable from Plan 01 migration)
       - phone: caller_phone
       - address: property_address or null
       - job_type: service_type
       - description: job_description
       - status: 'new'
       - source: 'voice_bot'
    8. Insert into leads with:
       - caller_name, caller_phone, caller_email
       - source_channel: 'voice_bot'
       - call_id: call_id
       - qualification_status: from payload or 'pending'
       - client_id: from step 6
       - quote_request_id: from step 7
    9. If sentiment is provided, update vapi_call_logs SET sentiment and outcome WHERE call_id matches. Also set lead_id to the newly created lead. This links the call log to the lead. Do not fail the request if this update finds no matching call_log row (the log may not exist yet if n8n calls intake before call logging completes).
    10. Return 201 with `{ success: true, quote_request_id, lead_id, client_id, client_created: boolean }`.
    11. Wrap all database operations in a try/catch. Return 500 with `{ error: message }` on failure. Log error details to console for Supabase function logs.

    Create a `json()` helper function matching the project pattern.

    Important: Do NOT use the anon key. Do NOT skip CORS. Do NOT use `Authorization: Bearer` for auth (n8n uses the custom x-webhook-secret header). Phone normalization must handle E.164 format (+17575551234) that Vapi sends.

    THEN create `supabase/functions/intake-lead/index.spec.ts` with Deno unit tests. Per CLAUDE.md TDD mandate, co-locate tests alongside source. Use `Deno.test()` and `assertEquals`/`assert` from `https://deno.land/std@0.190.0/testing/asserts.ts`. Import the exported pure functions from `./index.ts`.

    Test cases (pure logic tests, no database mocking required):
    1. `validateWebhookSecret` returns false when header is null
    2. `validateWebhookSecret` returns false when header does not match secret
    3. `validateWebhookSecret` returns true when header matches secret
    4. `validateRequiredFields` returns valid=false with missing fields listed when caller_name is absent
    5. `validateRequiredFields` returns valid=false when multiple fields are missing (lists all missing)
    6. `validateRequiredFields` returns valid=true when all required fields are present
    7. `normalizePhone` strips +1 prefix and non-digits: "+17575551234" becomes "17575551234"
    8. `normalizePhone` handles already-clean numbers: "7575551234" stays "7575551234"
    9. `normalizePhone` strips parentheses and dashes: "(757) 555-1234" becomes "7575551234"
    10. OPTIONS request to handleRequest returns 200 with CORS headers (if handleRequest is exported)
    11. Non-POST request returns 405
