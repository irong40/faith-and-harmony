  <name>Task 1: Create wf5 n8n workflow JSON</name>
  <files>n8n-workflows/wf5-vapi-intake-pipeline.json</files>
  <action>
Create `n8n-workflows/wf5-vapi-intake-pipeline.json` following the exact JSON structure of wf1 and wf4 (same node format, connection format, settings, meta, tags).

The workflow has these nodes in order:

1. **Vapi Webhook** (n8n-nodes-base.webhook, typeVersion 2)
   - httpMethod: POST, path: "vapi-intake"
   - authentication: "headerAuth" (uses Generic Credential for X-Vapi-Secret)
   - responseMode: "responseNode" (so Respond Accepted sends the HTTP response)

2. **Respond Accepted** (n8n-nodes-base.respondToWebhook, typeVersion 1)
   - respondWith: "json"
   - responseBody returns `{ received: true, call_id: $json.body.message.call.id }`
   - This MUST fire immediately before any processing to prevent Vapi timeout retries

3. **Filter End of Call** (n8n-nodes-base.if, typeVersion 2)
   - Condition: `$json.body.message.type` equals `end-of-call-report`
   - TRUE branch continues to Extract Fields
   - FALSE branch goes to Skip Set node

4. **Skip** (n8n-nodes-base.set, typeVersion 3.3)
   - Sets result: "skipped: not end-of-call-report"
   - Terminal node for non end-of-call events

5. **Extract Fields** (n8n-nodes-base.code, typeVersion 2)
   - JavaScript code that extracts from `$('Vapi Webhook').item.json.body.message`
   - Primary source: `analysis.structuredData` fields
   - Fallback: `call.customer.number` for phone, `analysis.summary` for job_description
   - Builds intake payload matching intake-lead required fields: caller_name, caller_phone, service_type, job_description, call_id
   - Validates required fields and sets `_can_proceed` boolean and `_validation_errors` array
   - Use the exact code from RESEARCH.md Pattern 2, including null safety

6. **Check Valid** (n8n-nodes-base.if, typeVersion 2)
   - Condition: `$json._can_proceed` equals true
   - TRUE branch continues to Call Intake
   - FALSE branch goes to Build Error Email (missing required fields = admin alert)

7. **Call Intake Lead** (n8n-nodes-base.httpRequest, typeVersion 4.2)
   - method: POST
   - url: `$env.SUPABASE_URL` + `/functions/v1/intake-lead`
   - Headers: `x-webhook-secret` with value `$env.INTAKE_WEBHOOK_SECRET`, Content-Type application/json
   - Body: JSON stringify of `$json` (the extracted fields, minus internal _can_proceed and _validation_errors)
   - continueOnFail: true (CRITICAL so the downstream IF node can check status on 4xx/5xx)
   - timeout: 30000

8. **Check Status** (n8n-nodes-base.if, typeVersion 2)
   - Condition: `$response.statusCode` less than 300
   - TRUE branch goes to Log Success
   - FALSE branch goes to Build Error Email

9. **Log Success** (n8n-nodes-base.set, typeVersion 3.3)
   - Sets result: "Intake created", quote_request_id from response, lead_id from response

10. **Build Error Email** (n8n-nodes-base.code, typeVersion 2)
    - JavaScript that builds the Resend API payload
    - from: "n8n-alerts@sentinelaerial.com" (or noreply@sentinelaerial.com)
    - to: ["contact@sentinelaerial.com"]
    - subject: "ALERT: Vapi intake failed"
    - text body includes: call_id, caller_phone (if available), error details from either validation_errors or HTTP response body, timestamp
    - Return the Resend API request body as JSON

11. **Send Admin Alert** (n8n-nodes-base.httpRequest, typeVersion 4.2)
    - method: POST
    - url: "https://api.resend.com/emails"
    - Headers: Authorization "Bearer $env.RESEND_API_KEY", Content-Type application/json
    - Body: JSON stringify of the email payload from Build Error Email
    - This node is terminal (no downstream connections)

Connections (matching wf1/wf4 format):
- Vapi Webhook -> Respond Accepted (main[0][0])
- Vapi Webhook -> Filter End of Call (main[0][1]) — NOTE: wf1 uses parallel connections from webhook to both respond AND next node
- Filter End of Call TRUE -> Extract Fields
- Filter End of Call FALSE -> Skip
- Extract Fields -> Check Valid
- Check Valid TRUE -> Call Intake Lead
- Check Valid FALSE -> Build Error Email
- Call Intake Lead -> Check Status
- Check Status TRUE -> Log Success
- Check Status FALSE -> Build Error Email
- Build Error Email -> Send Admin Alert
