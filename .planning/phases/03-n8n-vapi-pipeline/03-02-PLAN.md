After the user confirms the workflow is imported, send test webhook payloads to validate both success and failure paths.

Test 1: Successful intake (happy path)

Use curl to POST a simulated Vapi end-of-call-report payload to the n8n webhook URL. The payload must match the Vapi webhook format with body.message wrapping.

```bash
curl -X POST "https://[webhook-url]/webhook/vapi-intake" \
  -H "Content-Type: application/json" \
  -H "X-Vapi-Secret: [secret-value]" \
  -d '{
    "message": {
      "type": "end-of-call-report",
      "call": {
        "id": "test-wf5-001",
        "customer": {
          "number": "+17575551234",
          "name": "Test Caller WF5"
        }
      },
      "analysis": {
        "summary": "Caller wants aerial photos of a residential property in Virginia Beach",
        "structuredData": {
          "caller_name": "Test Caller WF5",
          "caller_phone": "+17575551234",
          "caller_email": "test@example.com",
          "service_type": "re_standard",
          "job_description": "Aerial photos of a 3 bedroom house in Virginia Beach for real estate listing",
          "property_address": "123 Atlantic Ave, Virginia Beach, VA 23451",
          "preferred_date": "2026-04-01",
          "qualification_status": "qualified",
          "sentiment": "positive"
        }
      },
      "endedReason": "customer-ended-call"
    }
  }'
```

Expected: HTTP 200 from n8n with { received: true, call_id: "test-wf5-001" }.
Then check n8n execution history to confirm the workflow ran through to Log Success.
Then check Supabase to confirm a new quote_request was created with source "voice_bot" and name "Test Caller WF5".

Test 2: Non end-of-call event (should be skipped)

```bash
curl -X POST "https://[webhook-url]/webhook/vapi-intake" \
  -H "Content-Type: application/json" \
  -H "X-Vapi-Secret: [secret-value]" \
  -d '{
    "message": {
      "type": "status-update",
      "call": { "id": "test-wf5-002" },
      "status": "in-progress"
    }
  }'
```

Expected: HTTP 200 ACK response. Workflow execution in n8n shows it reached "Skip" node (not end-of-call-report).

Test 3: Missing required fields (should trigger admin alert)

```bash
curl -X POST "https://[webhook-url]/webhook/vapi-intake" \
  -H "Content-Type: application/json" \
  -H "X-Vapi-Secret: [secret-value]" \
  -d '{
    "message": {
      "type": "end-of-call-report",
      "call": {
        "id": "test-wf5-003",
        "customer": {}
      },
      "analysis": {
        "summary": "Short call, caller hung up quickly",
        "structuredData": null
      },
      "endedReason": "customer-ended-call"
    }
  }'
```

Expected: HTTP 200 ACK. Workflow execution reaches Build Error Email and Send Admin Alert. Admin receives an email at contact@sentinelaerial.com with subject "ALERT: Vapi intake failed" indicating missing required fields.

Test 4: Unauthorized request (wrong secret)

```bash
curl -X POST "https://[webhook-url]/webhook/vapi-intake" \
  -H "Content-Type: application/json" \
  -H "X-Vapi-Secret: wrong-secret-value" \
  -d '{"message":{"type":"end-of-call-report"}}'
```

Expected: HTTP 403 or 401 from n8n (headerAuth rejects the request). No workflow execution.

- Delete clients where phone = '+17575551234' if no other data references them
