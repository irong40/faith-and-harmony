Create `availability_slots` table with columns:

- created_at (timestamptz, default now, NOT NULL)

- day_of_week (smallint, NOT NULL, CHECK 0 to 6). Add a SQL comment: "0=Sun, 1=Mon, 2=Tue, 3=Wed, 4=Thu, 5=Fri, 6=Sat. Matches PostgreSQL EXTRACT(DOW) and JavaScript getDay()."

- "admins_all" FOR ALL TO authenticated USING (has_role(auth.uid(), 'admin'::app_role)) WITH CHECK (has_role(auth.uid(), 'admin'::app_role))

- id (uuid, gen_random_uuid PK)
- created_at (timestamptz, default now, NOT NULL)
- updated_at (timestamptz, default now, NOT NULL)
- override_date (date, NOT NULL)
- is_available (boolean, NOT NULL). true = extra open day, false = closed override.
- note (text, nullable)
- service_type (text, nullable)

Enable RLS with same two policies (service_role_all, admins_all).
Add index on override_date.
Add COMMENT ON TABLE for both tables.

Create the availability-check edge function following the exact pattern from supabase/functions/pricing-lookup/index.ts (same import style, CORS headers, json helper, exported handleRequest, serve at bottom).

**index.ts:**

Import serve from deno std 0.190.0. Import createClient from supabase-js 2.45.0.

Set up CORS headers (same as pricing-lookup: Allow-Origin *, Allow-Headers with authorization/x-client-info/apikey/content-type, Allow-Methods GET/OPTIONS).

Create a json helper function (same as pricing-lookup).

Create a formatReadable function that takes an ISO date string (YYYY-MM-DD) and returns a human-readable string like "Monday March 10" using toLocaleDateString with timeZone 'America/New_York'. Parse dates at noon UTC (append T12:00:00Z) to avoid edge-of-day timezone shifts.

Export async function handleRequest(req: Request):
1. Handle OPTIONS (CORS preflight).
2. Reject non-GET methods with 405.
3. Parse query params: service_type (optional), start_date (required, YYYY-MM-DD), end_date (required, YYYY-MM-DD).
4. Return 400 if start_date or end_date missing.
5. Create Supabase client using SUPABASE_URL and SUPABASE_SERVICE_ROLE_KEY from env.
6. Query availability_slots for active slots. If service_type provided, filter with .or() for matching service_type or null. If no service_type, filter for null only.
7. Query availability_overrides in the date range.
8. Query blackout_dates in the date range.
9. Build a Set of active day-of-week numbers, a Map of override_date to is_available, and a Set of blackout dates.
10. Loop from start_date to end_date (use noon UTC to avoid DST issues). For each date:
    - Skip if in blackout set.
    - If override exists for this date, use override's is_available value.
    - Otherwise, check if day_of_week is in active days set.
    - If available, add to result array.
11. Format available dates into readable strings using formatReadable.
12. Return JSON with available_dates (ISO strings array), readable_dates (comma-joined human string or "No dates are currently available in that range"), and count.

Call serve(handleRequest) at the bottom.

Deploy with: `npx supabase functions deploy availability-check --no-verify-jwt`

Use --no-verify-jwt because the Vapi bot calls this endpoint without a Supabase session (same pattern as pricing-lookup).
