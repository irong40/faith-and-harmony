**src/hooks/useAvailability.ts:**

Create TanStack Query v5 hooks following the patterns used in existing admin pages. Import useQuery, useMutation, useQueryClient from @tanstack/react-query and supabase from @/integrations/supabase/client.

Hooks to create:

1. `useAvailabilitySlots()` - useQuery fetching all rows from availability_slots ordered by day_of_week. queryKey: ['availability_slots'].

2. `useBlackoutDates(startDate: string, endDate: string)` - useQuery fetching blackout_dates in a date range, ordered by blackout_date. queryKey: ['blackout_dates', startDate, endDate]. Use .gte('blackout_date', startDate).lte('blackout_date', endDate).

3. `useAvailabilityOverrides(startDate: string, endDate: string)` - useQuery fetching availability_overrides in a date range. queryKey: ['availability_overrides', startDate, endDate].

4. `useAddBlackoutDate()` - useMutation inserting into blackout_dates with { blackout_date, reason }. onSuccess invalidates queryKey ['blackout_dates']. onError shows a toast via sonner ("Failed to add blackout date" or if unique constraint "That date is already blocked").

5. `useRemoveBlackoutDate()` - useMutation deleting from blackout_dates by id. onSuccess invalidates ['blackout_dates'].

6. `useToggleSlot()` - useMutation updating availability_slots set is_active to the new value by id. onSuccess invalidates ['availability_slots'].

Export all hooks. Use TypeScript types for the table row shapes (define locally in the file, do not import from generated Supabase types since these tables are new and types may not be regenerated yet).

**src/pages/admin/Scheduling.tsx:**

Create the admin page following the structure of QuoteRequests.tsx: AdminNav at top, then content wrapped in Cards.

Layout with two Tabs (using Shadcn Tabs component):
- "Calendar" tab (default): Shows the Shadcn Calendar component with modifiers for color coding. Below the calendar, show a legend explaining the colors.
- "Weekly Slots" tab: Shows a table/list of the 7 days of the week (Sunday through Saturday) with each row showing day name, start_time, end_time, and an active/inactive toggle (Switch or Button).

Calendar tab details:
- Use the existing Calendar component from @/components/ui/calendar with mode="single".
- Compute the current month range for queries (first day of current month to last day of next month, giving a 2-month view).
- Use useAvailabilitySlots, useBlackoutDates, and useAvailabilityOverrides hooks.
- Build modifier arrays:
  - available: dates where day_of_week is in active slots and no blackout/negative override
  - blackout: dates from blackout_dates table
  - overrideOff: dates with negative override (is_available = false)
  - overrideOn: dates with positive override on a normally closed day
- Apply modifiersClassNames:
  - available: 'bg-green-100 text-green-800 rounded-md'
  - blackout: 'bg-red-100 text-red-600 line-through rounded-md'
  - overrideOff: 'bg-orange-100 text-orange-700 rounded-md'
  - overrideOn: 'bg-blue-100 text-blue-700 rounded-md'
- When a date is clicked, show a Dialog (Shadcn Dialog) with options:
  - If the date is available: offer "Block this date" with a reason input (text input with suggested values: weather hold, holiday, maintenance, personal) and a "Block Date" button.
  - If the date is blacked out: show the reason and offer "Remove Blackout" button.
- Use sonner toast for success/error feedback on mutations.
- Disable past dates on the calendar (dates before today).

Weekly Slots tab details:
- Display a Card for each day (Sunday through Saturday).
- Each card shows: day name (from a DAY_NAMES constant array), start_time, end_time, and a toggle for is_active.
- If no slot exists for a day (like Saturday/Sunday by default), show it as "No slot configured" with the toggle off.
- When the toggle changes, call useToggleSlot mutation.
- Use the Shadcn Switch component for the toggle.

Import CalendarDays from lucide-react for the page icon.

IMPORTANT: When converting ISO date strings from Supabase (YYYY-MM-DD) to Date objects for calendar modifiers, append 'T12:00:00' to avoid off-by-one timezone issues (research pitfall 2).

IMPORTANT: Do NOT use `onSuccess` in useQuery options (deprecated in TanStack Query v5). Only use callbacks in useMutation.
