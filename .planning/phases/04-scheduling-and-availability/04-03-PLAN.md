**src/vapi/system-prompt-additions.md:**

Create a markdown file containing the system prompt additions that should be appended to the main Paula bot system prompt (from Phase 2). These additions tell the bot how to handle availability queries.

Content should include:

1. **When to check availability:** When the caller asks about scheduling, booking, availability, "when can you come out," "what days are open," or similar.

2. **How to use the tool:** Call check_availability with today's date as start_date and 14 days out as end_date. If the caller specifies a date range, use those dates instead. If the service type is known from earlier in the conversation, include it.

3. **How to speak the results:** Use the readable_dates field from the response. Say something like "I have openings on [readable_dates]. Would any of those work for you?" Do NOT read ISO date strings (like 2026-03-10) to the caller.

4. **If no dates available:** Say "I don't have any openings in that timeframe. Would you like me to check the following week?" and retry with the next 14-day window.

5. **Capturing preferred date:** Once the caller picks a date, confirm it back to them and note it as their preferred_date in the call summary. Say "I've noted [day name and date] as your preferred date. We'll confirm that when we follow up with your quote."

6. **Do NOT promise a booking:** The bot does not book the job. It captures the preferred date and includes it in the intake data. The admin reviews and confirms scheduling.

Add a comment at the top of the file: "These additions should be appended to the main system prompt in the Vapi assistant configuration. They supplement the Phase 2 system prompt with scheduling behavior."
