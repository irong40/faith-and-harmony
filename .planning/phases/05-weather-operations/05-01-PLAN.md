  <action>
    Create the edge function following the tfr-refresh pattern (serve() from deno std, createClient from esm.sh, corsHeaders, fail-open error handling).

    The function does four things in sequence:

    1. FETCH NWS DATA
       - GET https://api.weather.gov/gridpoints/AKQ/90,52
       - Headers: User-Agent 'SentinelAerial/1.0 admin@faithandharmonyllc.com', Accept 'application/geo+json'
       - AbortSignal.timeout(20000)
       - Parse response.properties to get windSpeed, windGust, visibility, ceilingHeight, probabilityOfPrecipitation, skyCover, temperature time series

    2. EXPAND AND CONVERT
       - Implement parseDurationHours() to parse ISO 8601 durations like "PT3H" and "P1DT2H" into integer hours. Use regex: /P(?:(\d+)D)?T?(?:(\d+)H)?/
       - Implement expandToHourly() to expand NWS compressed time series into one value per hour for 48 hours starting from current hour (rounded down). Loop for h in [0, durationHours) (exclusive of endpoint per research pitfall 3).
       - Apply unit conversions at build time:
         - windSpeed/windGust: km/h to m/s (multiply by 1/3.6)
         - visibility: meters to statute miles (multiply by 0.000621371)
         - ceilingHeight: meters to feet (multiply by 3.28084, round to integer)
         - precipitationProbability: no conversion (integer percent)
         - temperature: no conversion (already Celsius)
       - When ceilingHeight is null, pass null (not 0) to evaluation. Null means unlimited ceiling (clear skies), not zero ceiling.

    3. EVALUATE AND CACHE
       - Port the evaluateWeather() logic verbatim from src/lib/weather-evaluation.ts into the edge function as evaluateWeatherInline(). Do NOT import from @/ paths (they do not resolve in Deno). Define WeatherThreshold and WeatherMetrics types inline.
       - Fetch all rows from weather_thresholds table using service role client.
       - For each of 48 hours, build a metrics object, run evaluateWeatherInline(), build a cache row.
       - Upsert all 48 rows into weather_forecast_cache with onConflict: 'forecast_hour'. Include fetched_at as the current timestamp.

    4. FLAG SCHEDULED JOBS
       - Query drone_jobs for jobs where scheduled_date is within the next 48 hours AND status is NOT in ('delivered', 'cancelled').
       - For each job, find the forecast row closest to the job's scheduled_date + scheduled_time (or use the start of the scheduled_date if no time).
       - If the determination for that hour is NO_GO or CAUTION, update the job's weather_hold to true and weather_hold_reasons to the determination_reasons array.
       - If the determination is GO, update weather_hold to false and weather_hold_reasons to null.

    Error handling: Follow the tfr-refresh fail-open pattern. Wrap the entire handler in try/catch. On error, log to console.error and return HTTP 200 with { ok: false, error: message }. This prevents pg_cron from raising alarms on transient NWS failures.

    Response on success: JSON { ok: true, hours: 48, jobs_flagged: N, fetched_at: ISO string }

    CORS headers: same pattern as tfr-refresh (Allow-Origin: *, handle OPTIONS preflight).
  </action>
