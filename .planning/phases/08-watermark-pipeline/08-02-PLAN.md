Two deliverables in this task.

DELIVERABLE A: Watermark tile PNG

Create a tiled diagonal watermark PNG programmatically using the `canvas` npm package (already used in Phase 7 for PWA icon generation). The watermark specifications from research:

- Canvas size: 1200x800 pixels
- Background: Fully transparent (alpha = 0)
- Text: "SENTINEL AERIAL" repeated in a diagonal tiled pattern
- Text color: White with 35% opacity (rgba 255, 255, 255, 0.35)
- Rotation: 30 to 45 degrees diagonal
- Font: Bold sans serif, approximately 48px
- Spacing: Text repeated every ~300px horizontally and ~200px vertically to create a grid pattern
- Drop shadow: 2px offset in dark gray (rgba 0, 0, 0, 0.15) for visibility on light areas

The tile pattern must cover the entire 1200x800 canvas so that when composited over any preview image, no region is free of watermark text. This prevents cropping recovery.

Write a Node.js script (can be a temp script, does not need to be permanent) that generates the PNG and saves to `assets/watermark-tile.png`. Run the script. Verify the file exists and is a valid PNG.

DELIVERABLE B: n8n WF1 Pipeline Update

Read the existing `n8n-workflows/wf1-sentinel-pipeline-orchestrator.json`. Understand its node structure and connections.

Add a new watermark_preview step consisting of these n8n nodes, inserted AFTER the qa_gate step and BEFORE the packaging step:

Node 1: "Select Preview Images" (Code node)
- Fetches processed assets for the current job from Supabase (using HTTP Request to `/rest/v1/drone_assets?mission_id=eq.{job_id}&processing_status=in.(processed,exported)&file_type=eq.image&order=qa_score.desc&limit=3`)
- If no processed images found, skip watermark step (output empty array, connect to packaging directly)
- Output: array of up to 3 asset objects with file paths

Node 2: "Download Original" (HTTP Request node, loop)
- For each selected asset, download the original from drone-jobs bucket using service key
- URL: `{{ $env.SUPABASE_URL }}/storage/v1/object/drone-jobs/{{ $json.source_path }}`
- Headers: Authorization Bearer service key
- Response type: Binary

Node 3: "Resize and Watermark" (Edit Image node or Execute Command node)
- If using Edit Image node: Resize to max width 1200px, then composite with watermark-tile.png
- If Edit Image composite is not reliable: Use Execute Command node with ImageMagick commands:
  ```
  magick input.jpg -resize 1200x -strip preview_base.jpg
  magick preview_base.jpg /path/to/watermark-tile.png -gravity center -compose dissolve -define compose:args=35 -composite preview_final.jpg
  ```
- CRITICAL: Include `-strip` flag to remove EXIF/GPS metadata from previews
- Output: Binary JPEG data

Node 4: "Upload Preview" (HTTP Request node, loop)
- Upload each watermarked JPEG to Supabase Storage
- URL: `{{ $env.SUPABASE_URL }}/storage/v1/object/watermark-previews/{{ $json.job_id }}/preview_0{{ $json.index }}.jpg`
- Method: POST
- Headers: apikey and Authorization Bearer service key, Content-Type image/jpeg
- Body: Binary data from previous node
- Use upsert header `x-upsert: true` to allow re-running the pipeline

Node 5: "Save Preview URLs" (HTTP Request node)
- PATCH `{{ $env.SUPABASE_URL }}/rest/v1/drone_jobs?id=eq.{{ $json.job_id }}`
- Body: `{ "preview_urls": [array of public URLs] }`
- Public URL format: `{{ $env.SUPABASE_URL }}/storage/v1/object/public/watermark-previews/{{ job_id }}/preview_01.jpg`

Connect the output of the watermark step to the existing packaging step. Ensure the pipeline continues to packaging/delivery regardless of whether watermark step produced 0, 1, 2, or 3 previews.

IMPORTANT: The n8n workflow JSON is the source of truth that gets imported into n8n. Update the JSON file in the repo. The user will import the updated workflow into their running n8n instance.

The watermark-tile.png file path in the Execute Command node should reference a path on the processing rig where the asset will be placed. Use `/opt/sentinel/assets/watermark-tile.png` as the reference path and note in the summary that the user needs to copy the asset to that location on the rig (or update the path in n8n to match where they place it).
