  <action>
Create `src/components/admin/LeadDetailDrawer.tsx`.

Define a local type for the combined data the drawer needs:
```typescript
type LeadDetail = {
  lead: {
    id: string;
    caller_name: string;
    caller_phone: string;
    caller_email: string | null;
    source_channel: string;
    qualification_status: string;
    created_at: string;
  };
  callLog: {
    id: string;
    transcript: string | null;
    summary: string | null;
    recording_url: string | null;
    duration_seconds: number | null;
    outcome: string | null;
    started_at: string | null;
  } | null;
};
```

The component accepts `{ leadId: string | null; onClose: () => void }` as props.

When `leadId` is null, render null (drawer stays closed via Sheet open={false}).

Fetch data with useQuery keyed on `["lead-detail", leadId]` with `enabled: !!leadId`. The query function:
1. Fetches the lead: `supabase.from("leads" as never).select("id, caller_name, caller_phone, caller_email, source_channel, qualification_status, created_at").eq("id", leadId).single()`
2. Fetches the call log: `supabase.from("vapi_call_logs" as never).select("id, transcript, summary, recording_url, duration_seconds, outcome, started_at").eq("lead_id", leadId).order("created_at", { ascending: false }).limit(1).maybeSingle()`
3. Returns `{ lead, callLog }` typed as `LeadDetail`.

Render a `<Sheet open={!!leadId} onOpenChange={(open) => !open && onClose()}>` wrapping a `<SheetContent side="right" className="w-full sm:max-w-2xl flex flex-col p-0">`.

Inside SheetContent, structure the layout with three sections:

Section 1 — Header (SheetHeader with px-6 pt-6 pb-4 border-b):
- SheetTitle showing `lead.caller_name`
- A row showing caller_phone and a source badge using SOURCE_CHANNEL_LABELS (not imported — inline the label map or accept a prop; keep it simple: use the raw source_channel value if no label map)
- Show a Skeleton loading state while isLoading is true

Section 2 — Call info (px-6 py-4 border-b, only if callLog exists):
- If callLog.summary exists: a `<div className="rounded-md bg-muted p-3 text-sm text-muted-foreground mb-3">` with the summary text
- If callLog.recording_url exists: a `<div className="mb-3">` with a `<p className="text-xs font-medium text-muted-foreground mb-1">Recording</p>` label and a native `<audio controls className="w-full h-10" src={callLog.recording_url} />` element. Do NOT open in a new tab — embed inline.
- If callLog.duration_seconds exists: show formatted duration using the same formatDuration logic from CallLogs.tsx (copy the function, do not import across pages)

Section 3 — Transcript (flex-1 min-h-0):
- ScrollArea wrapping a `<pre className="whitespace-pre-wrap font-mono text-sm p-6">` with the transcript text
- If no transcript: centered muted text "No transcript available"
- If no callLog at all: centered muted text "No call log found for this lead"

Handle the loading state by showing a Skeleton placeholder for each section. Handle errors with a simple error message in place of content.

Note for Plan 03: Leave a clearly commented `{/* PLAN-03: notes form will go here */}` placeholder between Section 2 and Section 3, and a `{/* PLAN-03: activity timeline will go here */}` at the bottom of the ScrollArea area (below the transcript pre).
