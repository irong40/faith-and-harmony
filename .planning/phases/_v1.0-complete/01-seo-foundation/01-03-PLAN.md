  <name>Task 1: Create sitemap.xml and update robots.txt</name>
  <files>
    public/sitemap.xml
    public/robots.txt
  </files>
  <action>
    Create public/sitemap.xml:

    The sitemap lists only the public landing page. Admin, pilot, and auth routes are private and must not appear in the sitemap.

    ```xml
    <?xml version="1.0" encoding="UTF-8"?>
    <urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
      <url>
        <loc>https://sentinelaerial.faithandharmonyllc.com/</loc>
        <lastmod>2026-02-26</lastmod>
        <changefreq>monthly</changefreq>
        <priority>1.0</priority>
      </url>
    </urlset>
    ```

    Replace public/robots.txt entirely with the following content. The existing file allows all routes to all crawlers with no blocks and no Sitemap directive — this must be replaced:

    ```
    User-agent: *
    Disallow: /admin/
    Disallow: /pilot/
    Disallow: /auth
    Disallow: /proposal/
    Disallow: /invoice/
    Disallow: /drone-upload/
    Disallow: /my-jobs/

    Sitemap: https://sentinelaerial.faithandharmonyllc.com/sitemap.xml
    ```

    Do not keep per-bot Allow rules from the old file. A single User-agent: * block with explicit Disallows is cleaner and correct.
  </action>
  <verify>
    <automated>npm run dev &amp; sleep 3 &amp; curl -s http://localhost:8080/robots.txt | grep -c "Disallow" &amp; curl -s http://localhost:8080/sitemap.xml | grep -c "urlset"</automated>
    <manual>Open http://localhost:8080/robots.txt in browser. Confirm Disallow: /admin/ is present and Sitemap: directive appears at the bottom. Open http://localhost:8080/sitemap.xml and confirm it renders as valid XML with the landing page URL.</manual>
  </verify>
  <done>robots.txt contains at minimum 3 Disallow lines (admin, pilot, auth) and a Sitemap directive. sitemap.xml is valid XML containing the production landing page URL.</done>
