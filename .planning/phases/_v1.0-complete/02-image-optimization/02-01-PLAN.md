Compress sentinel-logo.png from 1.06 MB to under 100 KB, generate a WebP variant, and add explicit width and height attributes to the logo img element.

Purpose: A 1.06 MB logo blocks first paint. Compressing it removes the largest single performance bottleneck on the landing page. WebP delivers an additional 30 to 40 percent size reduction for browsers that support it.
Output: Compressed sentinel-logo.png under 100 KB, sentinel-logo.webp, and updated LandingPage.tsx with width/height on the logo img tag.
</objective>

<task type="auto">
  <name>Task 2: Add width and height to the logo img tag</name>
  <files>src/pages/LandingPage.tsx</files>
  <action>
    Open src/pages/LandingPage.tsx.

    Find the logo img element at approximately line 16:
    ```tsx
    &lt;img src="/assets/landing/sentinel-logo.png" alt="Sentinel Aerial Inspections Logo" /&gt;
    ```

    The CSS in landing.css renders this image at `height: 80px; width: auto`. The compressed PNG is 400px wide. The natural aspect ratio determines displayed width. Add the natural dimensions of the compressed output as the width and height attributes. After compression to 400px wide, the height will be proportional to the original aspect ratio.

    To find the actual compressed dimensions, run:
    ```
    npx sharp-cli --input public/assets/landing/sentinel-logo.png metadata
    ```

    Use those exact pixel values (width and height from the metadata output) in the img tag. The result should look like:
    ```tsx
    &lt;img
      src="/assets/landing/sentinel-logo.png"
      alt="Sentinel Aerial Inspections Logo"
      width={400}
      height={N}
    /&gt;
    ```
    where N is the actual height returned by the metadata command.

    The CSS `height: 80px; width: auto` on `.lp-logo-section img` already controls the rendered display size. The width/height attributes are for layout shift prevention only. Do not remove or change the CSS.

    Do NOT wrap this in a picture element or add a srcset. The WebP variant is available but using it requires a picture element which is a larger change deferred to Phase 5. For now, just the PNG with width/height satisfies IMG-02 for the logo.
  </action>
  <verify>
