    The function accepts POST requests with a JSON body matching this shape:

    ```ts
    interface QuoteRequest {
      name: string;
      email: string;
      phone: string;
      service_type: string;
      preferred_date: string;
      message: string;
    }
    ```

    Validation: All fields except message are required. Return 400 with { error: "Missing required fields" } if any required field is absent or empty string.

    Email via Resend:
    - From: "Sentinel Aerial Inquiries <inquiries@sentinelaerial.com>"
    - To: ["contact@sentinelaerial.com"]
    - Reply-To: the submitter's email address
    - Subject: "New Quote Request: {service_type} from {name}"
    - Body: plain text listing all submitted fields clearly

    Plain text body format:
    ```
    New quote request submitted from the Sentinel website.

    Name: {name}
    Email: {email}
    Phone: {phone}
    Service Type: {service_type}
    Preferred Date: {preferred_date}
    Message: {message}

    Reply directly to this email to respond to the prospect.
    ```

    On success return 200 with { success: true }.
    On Resend failure return 500 with { error: "Failed to send notification" }.

    Include the standard CORS headers block that allows OPTIONS preflight. The from address "inquiries@sentinelaerial.com" is assumed to be verified in Resend (same domain as contact@sentinelaerial.com). If the domain is not verified, Resend will bounce. Note this in a comment in the function.

    Import Resend as: `import { Resend } from "npm:resend@2.0.0";`
    Use: `Deno.env.get("RESEND_API_KEY")`

    The component reads the URL hash on mount to pre-select service type. Use useEffect with window.location.hash to extract the service parameter:

    ```tsx
    import { useState, useEffect } from 'react';

    const SERVICE_OPTIONS = [
      { value: 'listing-lite', label: 'Listing Lite ($225)' },
      { value: 'listing-pro', label: 'Listing Pro ($450)' },
      { value: 'luxury-listing', label: 'Luxury Listing ($750)' },
      { value: 'construction-progress', label: 'Construction Progress ($450/visit)' },
      { value: 'commercial-marketing', label: 'Commercial Marketing ($850)' },
      { value: 'inspection-data', label: 'Inspection Data ($1,200)' },
    ];
    ```

    Hash parsing on mount:
    ```tsx
    useEffect(() => {
      const hash = window.location.hash;
      // Hash format from pricing CTAs: #quote?service=listing-lite
      if (hash.startsWith('#quote?service=')) {
        const serviceValue = hash.replace('#quote?service=', '');
        const match = SERVICE_OPTIONS.find((opt) => opt.value === serviceValue);
        if (match) setFormData((prev) => ({ ...prev, service_type: serviceValue }));
      }
    }, []);
    ```

    Form state and submission:

    - useState for formData (name, email, phone, service_type, preferred_date, message)
    - useState for status: 'idle' | 'submitting' | 'success' | 'error'
    - On submit, prevent default, set status to 'submitting'
    - POST to `${import.meta.env.VITE_SUPABASE_URL}/functions/v1/quote-request` with Content-Type application/json and body JSON.stringify(formData)
    - Include header: `apikey: ${import.meta.env.VITE_SUPABASE_ANON_KEY}` (same pattern used elsewhere in the app for public edge function calls)
    - On 200: set status to 'success'
    - On non-200 or network error: set status to 'error'

    When status is 'success', render an inline confirmation div instead of the form:
    ```tsx
    <div className="lp-quote-confirmation">
      <h3>REQUEST RECEIVED</h3>
      <p>Your quote request was submitted. Expect a response within one business day. Check your email for confirmation.</p>
    </div>
    ```

    When status is 'error', show an error message above the submit button:
    ```tsx
    <p className="lp-quote-error">Something went wrong. Call 760.575.4876 or email contact@sentinelaerial.com directly.</p>
    ```
